
архитектура генератора логов

1) логи должны быть воспроизводимы, для этого в параметре запуска переедавайте seed для random
2) надо умееть генерировать каждую команду из logs.txt
3) логи должны генерироваться без задержек -- задержки надо реализовывать current_time + rand_delay
т.е. не надо по настоящему ждать пока пройдет минута -- проще всего поддерживать shift

4) но так же надо поддерживать функцию -- сгенерить по 100 логов каждую секунду

реализация:

1) реализовать логи для всех сервисов (сервисов всего 5)
2) реализовать композицию логов -- решается через п.1
3) реализовать генератор
3.1) сгенерить Н логов выключиться
3.2) генерить Н логов кажду секунду в течении М секунду

4) в конце должен получиться исполняемый файл (желательно на питоне), принимающая аргументы и выполняющая дейсвтвия выше
